# Skills Tools

Skills are reusable knowledge packages that provide domain-specific instructions the model can activate when relevant.

OpenCode now supports **native skills** which are auto-discovered from your project's `.opencode/skill/` directory.

## Native Skill Syntax

To use a skill, simply use the native `use skill` command in your prompt:

```bash
use skill <name>
```

**Example:**
```bash
use skill tdd
use skill testing-patterns
```

---

## Authoring Tools

The following tools are used to create and maintain skills in your project.

## skills_create

Create a new skill with SKILL.md template.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | ✅ | Skill name (lowercase, hyphens) |
| `description` | string | ✅ | What the skill does and when to use it |
| `body` | string | ✅ | Markdown content with instructions |
| `tags` | string[] | ❌ | Tags for categorization |

**Example:**

```typescript
skills_create({
  name: "api-design",
  description: "REST API design patterns and best practices",
  body: "# API Design Guidelines\n\n1. Use nouns for resources...",
  tags: ["architecture", "api"]
})
```

---

## skills_init

Initialize a new skill with full directory structure and templates.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | ✅ | Skill name |
| `description` | string | ❌ | Initial description |

**Example:**

```typescript
skills_init({ name: "frontend-patterns" })
// Creates .opencode/skill/frontend-patterns/ directory with SKILL.md, scripts/, and references/
```

## Legacy Tools (Deprecated)

The following tools are **deprecated** and will be removed in a future version. They now function as redirection stubs.

- `skills_list()` -> Use native discovery
- `skills_use(name)` -> Use `use skill <name>`
- `skills_read(skill, file)` -> Read files directly
- `skills_execute(skill, script)` -> Run scripts via Bash tool

## Skills Workflow

```typescript
// List available skills
skills_list()

// Load skill for current task
skills_use({
  name: "testing-patterns",
  context: "Breaking dependencies in AuthService"
})

// Read full skill content (including references)
skills_read({ name: "testing-patterns" })

// Create a project-specific skill
skills_create({
  name: "auth-patterns",
  description: "OAuth and JWT patterns for this project",
  scope: "project"
})

// Add a script to automate common tasks
skills_add_script({
  skill_name: "auth-patterns",
  script_name: "generate-token.ts",
  content: "// Script to generate JWT tokens for testing"
})
```

## When to Use Skills

- **Before unfamiliar work** - Check if a skill exists
- **When you need domain-specific patterns** - Load the relevant skill
- **For complex workflows** - Skills provide step-by-step guidance
- **To capture project-specific knowledge** - Create custom skills

### Skill Triggers (Auto-load these)

```
Writing tests?           → skills_use(name="testing-patterns")
Breaking dependencies?   → skills_use(name="testing-patterns")
Multi-agent work?        → skills_use(name="swarm-coordination")
Building a CLI?          → skills_use(name="cli-builder")
Making architecture decisions? → skills_use(name="system-design")
```
